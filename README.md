# 🌙 Lunar Nova - Markdownプロジェクト管理

Markdownベースのモダンなプロジェクト管理Webアプリケーション。ブラウザ上でMarkdownを入力・編集し、美しいダッシュボードで視覚的に管理できます。

## ✨ 主な機能

### 📝 Markdownエディター
- **リアルタイムプレビュー**: 入力と同時にMarkdownがレンダリングされます
- **シンタックスハイライト**: モノスペースフォントで読みやすい編集体験
- **プレビュー切替**: エディターのみ表示とデュアルペイン表示を切り替え可能
- **XSS保護**: DOMPurifyによるHTMLサニタイゼーション

### 📊 ダッシュボード
- **カード形式の表示**: プロジェクトを視覚的に整理
- **ステータス管理**: 計画中/進行中/完了/保留/アーカイブの5つのステータス
- **統計情報**: 総プロジェクト数、進行中、完了、アーカイブ数を一目で確認
- **タグ機能**: プロジェクトを分類・整理
- **アーカイブ機能**: 完了したプロジェクトをアーカイブして整理

### 🔍 検索・フィルター・ソート
- **全文検索**: プロジェクト名、内容、タグを横断検索
- **ステータスフィルター**: 特定のステータスのプロジェクトのみ表示
- **ソート機能**: 
  - 更新日時（新しい順/古い順）
  - 作成日時（新しい順/古い順）
  - タイトル（A-Z/Z-A）
  - ステータス順
- **アーカイブ表示切替**: アーカイブされたプロジェクトの表示/非表示

### 📥📤 エクスポート・インポート
- **JSON形式でエクスポート**: 全プロジェクトをJSON形式で保存
- **Markdown形式でエクスポート**: 全プロジェクトを1つのMarkdownファイルに統合
- **JSONインポート**: バックアップから復元、または他の環境からインポート
- **マージ/置き換えモード**: インポート時に既存データとマージするか置き換えるか選択可能

### ☁️ クラウド同期（Firebase対応）
- **Firebase統合**: Firebaseを設定してクラウド同期を有効化
- **自動同期**: プロジェクト保存時に自動的にクラウドと同期
- **複数デバイス対応**: 異なるデバイス間でプロジェクトを共有

### 🎨 テーマ切替
- **ダークモード**: 目に優しいダークテーマ（デフォルト）
- **ライモード**: 明るい環境に最適なライトテーマ
- **ワンクリック切替**: ヘッダーのアイコンで即座に切り替え
- **設定保存**: テーマ設定をブラウザに保存

### 🎯 モダンなデザイン
- **グラスモーフィズム**: 透明感のあるモダンなUI
- **ネオングラデーション**: 鮮やかなアクセントカラー
- **スムーズなアニメーション**: 流れるような背景アニメーションとホバーエフェクト
- **レスポンシブデザイン**: デスクトップ、タブレット、モバイル対応

### 💾 データ永続化
- **LocalStorage**: ブラウザにデータを自動保存（サーバー不要）
- **即座に保存**: プロジェクトの変更が即座に反映
- **バックアップ機能**: エクスポートでバックアップを作成

## 🚀 使い方

### 起動方法

1. **シンプルな方法（ブラウザで直接開く）**
   ```bash
   # プロジェクトディレクトリに移動
   cd /Users/kousukenakamura/.gemini/antigravity/playground/lunar-nova
   
   # index.htmlをブラウザで開く
   open index.html
   ```

2. **ローカルサーバーを使用（推奨）**
   ```bash
   # Python 3の場合
   python3 -m http.server 8000
   
   # ブラウザで開く
   # http://localhost:8000
   ```

### 基本操作

#### 新規プロジェクト作成
1. ヘッダーの「✨ 新規プロジェクト」ボタンをクリック
2. プロジェクト名を入力
3. ステータスを選択（計画中/進行中/完了/保留/アーカイブ）
4. タグを入力（カンマ区切り、例: `開発, デザイン, リサーチ`）
5. Markdownエディターでプロジェクトの詳細を記述
6. 「💾 保存」ボタンをクリック

#### プロジェクトの検索
- ダッシュボードの検索ボックスにキーワードを入力
- プロジェクト名、内容、タグから検索
- 「×」ボタンで検索をクリア

#### プロジェクトのフィルタリング
- **ステータスフィルター**: ドロップダウンから特定のステータスを選択
- **アーカイブ切替**: 📦ボタンでアーカイブプロジェクトの表示/非表示

#### プロジェクトのソート
- ソートドロップダウンから並び順を選択：
  - 更新日時（新しい順/古い順）
  - 作成日時（新しい順/古い順）
  - タイトル（A-Z/Z-A）
  - ステータス順

#### プロジェクトの編集
- ダッシュボードでプロジェクトカードをクリック
- または、カードの「✏️ 編集」ボタンをクリック

#### プロジェクトのアーカイブ
- プロジェクトカードの「📦 アーカイブ」ボタンをクリック
- アーカイブされたプロジェクトは統計から除外されます
- アーカイブ解除は同じボタンから可能

#### プロジェクトの削除
- プロジェクトカードの「🗑️ 削除」ボタンをクリック
- 確認ダイアログで「削除する」を選択

#### データのエクスポート
1. ヘッダーの「📥 エクスポート」ボタンをクリック
2. エクスポート形式を選択：
   - **全てをJSON形式**: 完全なデータ構造を保存（バックアップ推奨）
   - **全てをMarkdown形式**: 読みやすいドキュメントとして保存

#### データのインポート
1. ヘッダーの「📤 インポート」ボタンをクリック
2. JSONファイルを選択
3. インポートモードを選択：
   - **既存のプロジェクトとマージ**: 新しいプロジェクトのみ追加
   - **既存のプロジェクトを置き換え**: 全てのプロジェクトを置き換え
4. 「インポート」をクリック

#### テーマの切替
- ヘッダーの「🌙」（または「☀️」）ボタンをクリック
- ダークモード ⇔ ライトモード を即座に切り替え
- 設定は自動的に保存されます

#### Firebase同期の設定
1. ヘッダーの「☁️ 同期」ボタンをクリック
2. Firebase設定を入力：
   - API Key
   - Project ID
   - Auth Domain
3. 「保存して同期」をクリック
4. 以降、プロジェクト保存時に自動同期

#### キーボードショートカット
- `Ctrl/Cmd + S`: プロジェクトを保存（エディター画面）
- `Ctrl/Cmd + K`: 検索ボックスにフォーカス
- `Esc`: モーダル・ドロップダウンを閉じる

## 📊 データ構造

### プロジェクトオブジェクト
```json
{
  "id": "unique-id",
  "title": "プロジェクト名",
  "status": "active",

  "content": "Markdown内容",
```

### ステータスの種類
- `planning`: 計画中
- `active`: 進行中

## 🐛 トラブルシューティング

### プロジェクトが表示されない
- ブラウザのLocalStorageが有効か確認
- プライベートブラウジングモードを使用していないか確認
- ブラウザのコンソールでエラーを確認

### エクスポートが動作しない
- ブラウザがファイルダウンロードを許可しているか確認
- ポップアップブロッカーが有効でないか確認

### インポートに失敗する
- JSONファイルの形式が正しいか確認
- ファイルが破損していないか確認
- ブラウザのコンソールでエラーメッセージを確認

### Firebase同期ができない
- Firebase設定が正確か確認
- Firestoreのセキュリティルールが設定されているか確認
- ネットワーク接続を確認
